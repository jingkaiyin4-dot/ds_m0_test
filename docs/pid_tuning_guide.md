# 减速电机双环 PID 调试与优化指南

本指南旨在帮助您理解并调试基于 M0 核心的直流减速电机双环（位置环 + 速度环）PID 控制系统。文档记录了核心参数的含义、修改位置以及常见问题的排查方向。

## 一、系统架构简述

控制框架位于 `PID.c` 与 `main.c` 中，采用**定时器中断 (10ms)** 进行固定周期的闭环计算。

*   **内环（速度环）**：输入为“目标速度”，反馈为“编码器计算出的当前速度”，输出为“PWM 占空比（Duty）”。
*   **外环（位置环）**：输入为“目标位置”，反馈为“编码器累计脉冲数”，输出为“目标速度”。

整个系统的数据流：`目标位置 -> 位置环 PID -> 目标速度 -> 速度环 PID -> PWM占空比 -> 电机转动 -> 编码器反馈`

## 二、关键参数修改指南

所有核心的 PID 系数和电机参数都初始化在 `PID.c` 的 **`DriveController_Init()`** 函数中。如果在 `main.c` 的 `App_StartDrivePidTest()` 中进行了复写（Override），则以 `main.c` 为准。

### 1. 速度环（内环）参数
位置：`PID.c -> DriveController_Init()`
变量结构：`g_driveController.left.speed`

| 参数名 | 默认值 | 作用与调试经验 |
| :--- | :--- | :--- |
| `kp` | `1.8f` | **比例系数**。决定了对速度误差的响应力度。若电机转速跟不上目标，可适当增大；若电机出现高频抖动或啸叫，说明 `kp` 过大，需减小。 |
| `ki` | `0.11f` | **积分系数**。用于消除稳态误差。若发现设定速度为 2 RPS，但电机始终稳定在 1.8 RPS 上不去，说明需要增大 `ki`；若遇到起步时积分累加过高导致的超调（冲过头），可适当减小。 |
| `kd` | `0.015f`| **微分系数**。用于预测未来误差，抑制超调。由于低分辨率编码器在低速时会产生剧烈的量化噪声，**不建议将 `kd` 调得太大**，否则会放大噪声导致电机剧烈抖动。 |
| `outputLimit`| `100.0f`| 速度环的输出即为 PWM 占空比，因此最大限制为 100%。 |

### 2. 位置环（外环）参数
位置：`main.c -> App_StartDrivePidTest()` 或 `PID.c`
变量结构：`g_driveController.left.position`

| 参数名 | 默认值 | 作用与调试经验 |
| :--- | :--- | :--- |
| `kp` | `0.006f`| 决定接近目标位置时的减速平缓度。如果靠近目标时太慢，可增大；如果到达目标时刹不住车（冲过头），需减小。 |
| `ki` | `0.0f` | **（已优化）强烈建议保持为 0**。由于电机本身具有将速度积分成位置的物理特性，外环加上积分极易导致目标改变时的严重超调（来回震荡）。 |
| `kd` | `0.165f`| 提供“阻尼”效果，防止位置超调。如果到达目标位置时有来回摆动，可适当增大 `kd`；若过大，可能导致电机在接近目标时发出抖动声。 |
| `outputLimit`| `10.0f` | **极其重要**。这决定了电机在寻位过程中的**最大允许速度 (RPS)**。如果希望电机跑得快一点，可以调大（前提是速度环能跟上）；如果希望温柔地跑过去，调小该值即可。 |

### 3. 系统硬件参数
位置：`PID.c -> 宏定义区`

*   `MG513XP26_ENCODER_COUNTS_PER_MOTOR_REV` (11.0f): 电机轴一圈的脉冲数。**注意**：目前底层中断（`Encoder.c`）为1倍频采集。若以后改为四倍频（AB相双边沿触发），此值需改为 44.0f。
*   `MG513XP26_GEAR_RATIO` (26.0f): 减速比。
*   `CONTROL_SAMPLE_PERIOD_S` (0.01f): 控制周期（10ms）。若更改了定时器中断频率，必须同步修改此值，否则所有 RPS 换算都会出错。

## 三、进阶调试与修改建议

如果您以后遇到以下问题，请查阅对应的修改建议：

### Q1: 电机在起步或低速时有严重的“嗡嗡”声，且转动不连贯
*   **原因**：低分辨率编码器（11 PPR）在 10ms 周期内产生的脉冲极少，甚至有时是 0 有时是 1，这带来了巨大的速度量化误差。这个误差被速度环的 `kp` 和 `kd` 放大，导致了抖动。
*   **修改函数**：`PID.c -> WheelController_Update()`
*   **如何修改**：
    1.  调节其中的**一阶低通滤波系数**：`wheel->speedRps = 0.3f * rawSpeedRps + 0.7f * wheel->speedRps;`。将 `0.3f` 减小（例如改为 `0.1f`）可以使滤波更平滑，但响应会稍微变慢。
    2.  如果还是抖动，尝试把速度环的 `kd` 设为 `0`，仅使用 PI 控制。

### Q2: 改变目标位置时，起步非常突兀或有撞击感
*   **原因**：目标位置瞬间发生跳变（例如直接加了 3 圈），导致误差 `error` 瞬间变大。
*   **修改函数**：不要直接在 `main.c` 设定目标，而是自己编写一个轨迹规划函数。
*   **如何修改**：不要用 `g_driveController.left.position.target = 858;` 这种一刀切的方式。可以写一个定时器逻辑，每 10ms 将 `target` 增加一个步进值，直到加到 858。这被称为**梯形速度规划**，能让起步和停车都如丝般顺滑。

### Q3: 停车时，电机总是有一定的误差（例如停不到完美的 0 点），但是又不动了
*   **原因**：电机存在机械死区（静摩擦力）。当位置极度接近目标时，PID 算出来的占空比可能只有 5%，不足以克服静摩擦让电机转动。
*   **修改函数**：`Motor.c -> Motor_SetDuty()`
*   **如何修改**：加入死区补偿。找出电机刚好能转动的最小占空比（比如 10%），然后这样改：
    ```c
    if (duty > 0) {
        duty += 10; // 死区补偿
    } else if (duty < 0) {
        duty -= 10;
    }
    ```

### Q4: 如何防止微分突变（Derivative Kick）？
在目前的 `PID.c -> PIDController_Update()` 中，我们已经使用了 `derivative = -(actual - pid->prevActual);` 代替了原来的误差微分。这是为了防止目标值（Target）被您用代码强行改动时，微分项产生剧烈尖峰。**今后请不要将该行改回 `error - prevError`**，除非是在一个目标值永远不会突变的系统中。
